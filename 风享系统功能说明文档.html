一、系统概述

「风享」是一款仿小红书的图文/视频博客社区系统，支持用户发布笔记、社交互动、内容发现等功能，并提供完整的管理后台用于社区运营和内容风控。系统采用前后端分离架构，后端为 Spring Boot + MyBatis，前端为 Vue 3 + Element Plus。

项目说明
系统名称风享（FengShare）
后端框架Spring Boot 4.0.6 + MyBatis + MySQL 8.0
前端框架Vue 3.4 + Vite 5 + Element Plus 2.7 + Pinia 2
认证方式JWT（Bearer Token）
密码加密BCrypt
文件存储阿里云 OSS
数据库MySQL 8.0 · 15 张业务表

三、用户端功能 用户端

3.1 用户账号

四、管理端功能 管理端

管理端通过独立的路由（/admin/*）、布局组件（AdminLayout.vue）、API 模块（api/admin.js）与用户端完全隔离。管理员登录需选择「管理端」选项，后端校验 role=1 方可进入。

4.1 数据看板

模块功能说明
核心概览4 组数据卡片：用户数据（总数/今日新增/禁言/禁用）、内容数据（总帖/今日新增/图文档/视频数/违规下架）、今日互动（点赞/评论/收藏）、举报数据（今日新增/待处理/处理率/通过率）
数据趋势近 7 天 / 30 天切换：用户新增趋势、帖子发布趋势、互动量趋势、举报量趋势
排行榜4 个维度 TOP 20：热门帖子、活跃创作者、热门标签、高频违规用户

4.2 用户管理

功能说明
用户列表展示 ID/头像/用户名/账号状态/禁言状态/角色/注册时间，支持按用户名/手机/邮箱搜索，按状态/禁言/角色/注册时间筛选
批量操作批量启用/禁用账号、批量禁言/解禁
禁言管理弹窗设置禁言时长（1/3/7/30天/永久）+ 填写原因，系统自动计算到期时间。支持手动提前解禁
禁言状态展示列表中标注「正常」「禁言中」「已到期」「永久禁言」四种状态，已到期用户可手动刷新或等待懒解封
禁言历史弹窗查看每个用户的完整禁言/解封记录（操作人、时间、时长、原因、解封方式）
重置密码管理员可强制重置任意用户密码
禁言流程： 管理员选择用户 → 设置时长+原因 → ├─ 写入 users 表禁言字段（is_muted=1, mute_end_time） ├─ 写入 user_mute_history 表（action=1） └─ 发送系统通知给被禁言用户 解禁： 手动解除：管理员操作 → 清除禁言字段 + 写入历史（action=2, unmuteType=1） 懒解封：用户发帖/评论时触发 → 系统自动清除 + 写入历史（action=2, unmuteType=2）

4.3 内容管理

功能说明
帖子列表展示 ID/封面/标题/作者/类型/状态/互动数据/发布时间。支持标题全文搜索、按状态/类型/日期范围筛选
帖子详情审核弹窗展示完整帖子内容（正文/图片网格/视频播放器/标签/位置），支持图片点击预览放大、视频播放控制（关闭弹窗自动停播）
审核操作审核通过（status: 2→1 公开可见）、违规下架（status→3 记录下架原因+通知作者）
批量操作批量下架/恢复/置顶/加精
物理删除彻底删除帖子及关联的图片/标签数据
状态日志每次状态变更写入 post_status_log 表（操作人/变更前/变更后/原因）
审核流程： 用户发布 → status=2（审核中，不公开） ↓ 管理员查看详情（图文网格 / 视频播放） ├─ 审核通过 → status=1（公开可见） └─ 违规下架 → status=3（记录下架原因 → 通知作者）

4.5 举报处理

功能说明
工单列表展示举报 ID/类型/对象摘要/举报人/原因/状态/处理人，按类型和状态筛选
统计卡片6 格数据概览：总数/待处理/已处理/驳回/平均处理时长/通过率
处理弹窗展示举报详情+被举报对象摘要。通过时选择联动处罚：下架帖子 / 删除评论 / 禁言用户（+时长）/ 禁用账号 + 填写处理备注。驳回时填写驳回原因。自动通知举报人处理结果
举报联动处罚： 管理员处理举报 → 通过 ├─ penaltyAction=1 → 下架帖子（post status→3 + 记录日志） ├─ penaltyAction=2 → 删除评论（comment status→0） ├─ penaltyAction=3 → 禁言用户（is_muted=1 + 记录历史 + 通知） └─ penaltyAction=4 → 禁用账号（user status=0 → JwtFilter 拦截） ↓ 通知举报人处理结果

五、处罚体系

5.1 三级处罚机制

级别处罚效果触发位置
1 级内容下架帖子/评论对用户不可见举报联动 / 管理端手动操作
2 级禁言（mute）禁止发帖和评论，支持定时到期 + 懒解封举报联动 / 管理端手动操作
3 级禁用账号无法登录、所有 API 返回 403举报联动 / 管理端手动操作

5.2 懒解封机制

工作原理： 1. 禁言到期后，系统不主动清除禁言状态（节省定时任务开销） 2. 用户下次发帖/评论时 → checkMuteStatus() 检测到 isMuted=1 且 muteEndTime < NOW() → ┌─ 自动清除禁言字段 ├─ 写入禁言历史（unmuteType=2 系统懒解封） └─ 正常处理请求 3. 管理端「禁言已到期」列表可查看待懒解封的用户，支持手动触发

5.3 拦截链路

请求 → JwtFilter ──token校验──→ 查 user.status ├─ status=1 → 放行 └─ status=0 → 403「账号已被禁用」 控制器 → Service.addPost / addComment ├─ isMuted=0 → 放行 ├─ isMuted=1 + 已到期 → 懒解封 → 放行 ├─ isMuted=1 + 未到期 → 拒绝 + 提示到期时间 └─ isMuted=1 + muteEndTime=NULL → 拒绝 + 提示永久禁言
